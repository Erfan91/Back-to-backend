# Back-to-Backend ⚙️
### A Node.js & Express Backend Showcase

A focused backend project demonstrating clean API architecture, MongoDB data modeling, and solid REST principles. Built to showcase backend fundamentals done right — no shortcuts.

---

## 🎯 Purpose

This project is a deliberate return to backend fundamentals. After working on full stack apps, I built this to demonstrate clean, production-ready backend code: well-structured routes, proper data modeling, and RESTful design throughout.

---

- 🔗 **REST API Design** — Clean, consistent endpoints following REST conventions
- 🗄️ **MongoDB & Mongoose** — Structured data models with validation and relationships
- 📦 **CRUD Operations** — Full Create, Read, Update, Delete across resources
- 🏗️ **MVC Architecture** — Separation of concerns with routes, controllers, and models

## 🚀 Getting Started

### Prerequisites
- Node.js >= 14
- MongoDB (local or Atlas)

### Installation

1. **Clone the repository**
```bash
git clone https://github.com/Erfan91/back-to-backend.git
cd back-to-backend
```

2. **Install dependencies**
```bash
npm install
```

3. **Set up environment variables**

Create a `.env` file in the root:
```env
PORT=5000
MONGO_URI=your_mongodb_connection_string
```

4. **Run the server**
```bash
npm run dev
```

Server runs at `http://localhost:5000`
